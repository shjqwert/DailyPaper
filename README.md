# 日报填写助手

每日工作记录录入、工时统计、自动填写公司内网日报/周报/月报。

---

## 文件结构

```
DailyPaper/
├── src/
│   ├── fill/
│   │   ├── base.py       # 公共登录逻辑
│   │   ├── daily.py      # 日报自动填写
│   │   ├── weekly.py     # 周报自动填写
│   │   └── monthly.py    # 月报自动填写
│   ├── db.py             # 数据库
│   ├── menu.py           # 交互菜单
│   └── report.py         # 功能逻辑
├── config.json           # 配置文件
├── data.db               # 本地数据库（自动生成）
├── setup.bat             # 安装依赖
└── 日报助手.cmd           # 启动入口
```

---

## 快速开始

### 第一步：安装依赖（只需一次）

双击运行 `setup.bat`，自动安装 Playwright。

### 第二步：配置选项

打开 `config.json`，填入实际内容：

```json
{
  "intranet": {
    "url": "http://公司内网地址",
    "username": "你的用户名",
    "password": "你的密码"
  },
  "options": {
    "项目": ["项目A", "项目B"],
    "节点": ["需求分析", "开发", "测试", "上线"],
    "类型": ["新增", "优化", "修复", "调研"],
    "模块": ["前端", "后端", "数据库", "接口", "其他"],
    "时间类型": ["正常", "加班", "出差"]
  }
}
```

## 功能说明

### 1 · 添加今日记录 / 2 · 补录历史日期记录

按数字键选择各字段，无需手动输入：

```
【项目】
  1. 项目A
  2. 项目B
  0. 手动输入
请选择编号: 1

【工时(小时)】请输入(如 2 或 1.5): 3

【工作内容】请简要描述: 完成登录模块开发

✓ 已保存: [项目A] 后端 - 完成登录模块开发 (3h)

继续添加? (y/n, 默认n):
```

### 3~5 · 查看明细

列出所选时间范围内的所有记录，显示全部字段及合计工时。

### 6~9 · 工时统计（按项目）

按项目汇总工时，带进度条和占比：

```
=== 按项目工时统计 本周 [2026-03-09 ~ 2026-03-15] ===

  项目A
    ████████████ 6h (60.0%)
      · 2026-03-10 后端 完成登录模块 (3h)
      · 2026-03-11 后端 接口联调 (3h)

  项目B
    ████████ 4h (40.0%)
      · 2026-03-12 前端 页面开发 (4h)

  总计: 10h
```

### 10~11 · 加班统计

按日汇总工时，超出 `config.json` 中 `standard_hours`（默认 8h）的部分计为加班，显示每日明细和累计加班时长。

### 12~14 · 自动填报

自动打开浏览器，登录内网，逐行填写表单，确认后提交。

> 使用前需先完成 **自动填报配置**（见下方）。

## 自动填报配置

> 需回公司连接内网后操作。

**操作步骤：**

1. 用浏览器打开内网填报页面
2. 按 `F12` 打开开发者工具
3. 右键点击要填写的输入框 → **检查**
4. 在高亮的 HTML 上右键 → **Copy → Copy selector**
5. 将复制的 selector 填入对应文件

**日报** → `src/fill/daily.py`

```python
SELECTORS = {
    "日期":     "#input-date",        # 粘贴实际 selector
    "项目":     "select[name='proj']",
    "工作内容": "textarea#content",
    "提交按钮": "button.submit",
    ...
}
```

**周报** → `src/fill/weekly.py`
**月报** → `src/fill/monthly.py`（同上，字段按实际页面调整）

配置完成后，运行菜单选 **12 / 13 / 14** 即可一键自动填报。

## 常见问题

**Q：选项里没有我的项目名怎么办？**
A：录入时选 `0` 手动输入，或直接在 `config.json` 的 `项目` 列表中添加。

**Q：填错了记录怎么办？**
A：选 **3 查看今日记录**，找到对应的 `id`，直接在命令行执行：
```
python src/report.py delete <id>
```
删除前会显示记录内容并要求二次确认，不会误删。

**Q：data.db 在哪里，能备份吗？**
A：在根目录 `DailyPaper/data.db`，直接复制即可备份。
